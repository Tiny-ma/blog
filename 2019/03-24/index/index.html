我对的前端职业的见解

前言

最近在找一些前端面试方面的问题，并不是要找工作，而是为公司面试准备题目。当然，这项任务对我来说是额外的，但我依然很感兴趣，也很感谢我的领导能给我这个机会。因为，这项额外的工作，能给我带来丰厚的回报—-是在未来的某天。

其实，从技术这块来说，我是个十足的菜鸟，菜中菜。诚然，相对的刚入门的人，还有稍微略强一点。我对自己的定位是，各个方面都懂一点，但没有一方面达到熟练，以至精通的。当然，这也是我未来的目标。

由于个人，平常爱遐想，下面也只是个人见解+吐槽，仅代表个人看法，肯定会有不对之处。望海涵。
本篇章不涉及技术教学，仅个人总结，对前端方面有个大体认知。

接下来，开始正题。

什么是程序员？

说到前端，就不得不提一下程序员。

程序员！百度百科上定义为：

程序员（英文Programmer）是从事程序开发、程序维护的专业人员。一般将程序员分为程序设计人员和程序编码人员，但两者的界限并不非常清楚，特别是在中国。软件从业人员分为初级程序员、中级程序员、高级程序员（现为软件设计师）、系统分析员，系统架构师，测试工程师六大类。

可以理解为从事程序开发的那群人。当然，也经常会被那些不懂行人的理解为，修电脑的之类的。但实际上，大多数程序员是不会修电脑的。

那么问题来了，程序员应不应该学会修电脑呢?

个人觉得应该会。首先，毕竟是我赖以生存的伙伴。日常维护的什么的，当然要由自己来做的比较好。其次，computer的不管是硬件，还是软件部分，对其运行原理的理解的，当然，不能指望自己能造出一个来，对其的理解， 对我们自己的提高，以后未来路程规划上，都会有很大帮助。

应该是所有程序员都知道的，所有的程序都是由0、1组成，表示电路的通断。但，0、1又是怎么储存在硬盘内存中的呢？CPU又是怎么如何计算的？

好吧，感觉这种事涉及过多，有时理解还真够难的。
都9102年了，我觉得吧，身为一个程序员，开发者，至少要知道：为什么会发明Computer这种东西，这东西是干嘛用的。以及自己写的程序是干嘛用的。

什么是前端？

前端这个职业，是最近几年才开始火起来的。为什么会火，我也不知道，是偶然？还是必然？

曾记得我在大学时，还没有前端这个职业，当初的学的.Net专业，感觉是跟着大佬混，工作不用问，而且，那时.Net的工作也确实多。然，最后几年不知道是啥情况了。。。额，跑题了。

前端开发工程师的可以说是程序员的一个子职业，2007年才真正开始受到重视的一个新兴职业，开发内容主要于浏览器页面。所以前端也叫web前端开发工程师。

Web前端开发技术是一个先易后难的过程，主要包括三个要素：HTML（标准通用标记语言下的一个应用）、级联样式表和JavaScript。

前端工程师的发展历史，可以简单概括如下：

  • 1990年，Tim Berners Lee发明了世界上第一个网页浏览器WorldWideWeb。
  • 1995年，Brendan Eich只用了10天便完成了第1版网页脚本语言（也就是目前我们所熟悉的JavaScript）的世界。
  • 2005年，AJAX技术的问世令静态的网页“动”了起来。
  • 2008年，Google推出了全新的JavaScript引擎V8.
  • 2009年，Node.js问世

正常前端涉及知识点

以前其实说了那长么多，其实都是废话。
前端的基础知识栈如下：

HTML

除了最基本的div、p、span、ul、li等之外。
还有HTML5标准的svg、canvas、Web Worker、WebSocket、Audio、Video等。
HTML部分，还有一点就是HTML结构，语义化标签。好的HTML标签，会让你的CSS很省很多事。

CSS

CSS部分，最重要部分，是布局。常见的布局有两栏、三栏、居中、
盒子模型、流式布局、响应式布局、flex弹性布局等。

BFC全称是Block Formatting Context，即块格式化上下文。

由有CSS3的盛行，高性能 CSS3 动画，也可说是前端一大亮点。

JavaScript

在JS部分，可以说是现在前端开发最重要，也是最难的一部分。

除基本JS语法外。JS重点有：

  • 闭包
  • 原型链
  • this、apply、call、bind
  • JS执行机制
  • JS新特性（ES6）

BOM

BOM是浏览对象，虽然前端是基于浏览器的，BOM提供了一些与浏览器交互的操作。

DOM

DOM是BOM的一个子对象，代表了我们可访问 JavaScript HTML 文档的所有元素。

当网页被加载时，浏览器会创建页面的文档对象模型（Document Object Model）。
HTML DOM 模型被构造为对象的树。

虽然现在MVC/MVVM/MVP各种思想下，现在框架提倡数据驱动。但DOM的基于操作，仍然是要重点掌握的内容。

通信

  • 跨域
  • AJAX
  • Fetch

安全

  • 攻击原理及防护措施

性能优化

  • HTML优化

不正常前端涉及知识点

上面的知识应该正常属于前端范畴。
下面应该属于整个开发行业。

面向对象

面向对象，三大特征是：封装、继承、多态。

面向对象的思想是，万事万物都是对象。任何都可以表示为一个对象。这些其实说着容易，作起来挺难的。尤其是对前端来说。毕竟前端的东西并只有JS而已。

设计模式

设计模式其实就是一些常见的问题解决方案，代表了最佳的实践。

设计模式是一套被反复使用的、多数人知晓的、经过分类编目的、代码设计经验的总结。

设计模式的六大原则

  • 开闭原则（Open Close Principle）
    开闭原则的意思是：对扩展开放，对修改关闭。

  • 里氏代换原则（Liskov Substitution Principle）
    里氏代换原则中说，任何基类可以出现的地方，子类一定可以出现。

  • 依赖倒转原则（Dependence Inversion Principle）
    针对接口编程，依赖于抽象而不依赖于具体。

  • 接口隔离原则（Interface Segregation Principle）
    使用多个隔离的接口，比使用单个接口要好。

  • 迪米特法则，又称最少知道原则（Demeter Principle）
    一个实体应当尽量少地与其他实体之间发生相互作用，使得系统功能模块相对独立。

  • 合成复用原则（Composite Reuse Principle）
    尽量使用合成/聚合的方式，而不是使用继承。

数据结构和算法

程序 = 数据结构 + 算法

数据结构就是计算机存储、组织数据的方式，它是人们为了实现各种各样的算法和程序而设计出来的，具有一定性质或规律。比如栈可以用来实现递归算法的非递归化，图可以用来处理网络问题。

算法你可以理解为用程序解决实际问题的方法。比如排序算法，查找算法。

程序的本质就是建立数据类型＋依靠数据类型和算法解决问题的。

总结

前端只是一个程序员的一个分支，用于主要处理浏览器相关程序的开发。

而程序员是使用PC开发程序，解决生活中的问题。

就技术方面，我们不能兼顾所有，专一而至。

就未来而言，我们不能拘泥一节，浏览器只是一种表现形式而已，毕竟现在浏览器火了，但谁知道会有多久呢，下一个火的是谁？VR? AR?

就现在而言，node.js摆在那里，前端已不只是仅仅是前端。
